## Vibe Coding Rules V2.7 — 自动触发器

> 以下内容由 install.bat 自动写入。删除本段即卸载触发器。

### 强制纪律（每次 skill 加载必执行，不可跳过）

**每次加载 skill 时必须先执行 Step -1（平台检测 → Memory自愈 → 项目状态检测），再进入后续流程。**

### 新项目检测

当你识别到用户要开新项目（说"我要做一个XX"、"新建项目"、"初始化项目"等），**必须询问**：

```
我检测到你要开始一个新项目，要不要初始化 AI 编程流水线？

它会帮你：
- 自动创建项目结构
- 搭上代码质量自检（改完代码自动检查常见错误）
- 搭上自动化测试
- 规则会从你的开发实践中自动生长进化

要初始化吗？（要 / 先不要 / 了解更多）
```

### 存量项目接管（V2.6 新增）

当你识别到已存在的项目**没有**开发文档目录或 `.ai-pipeline/` 配置时，必须询问：

```
我检测到这个项目还没有 AI 编程流水线，要不要现在搭上？

不需要你手动改代码——我会：
- 自动创建开发文档标准目录
- 扫描你现有的代码，把路由、接口、工具定义写入文档骨架
- 运行首次强制自检，建立代码质量基线

要搭上吗？（要 / 先不要 / 了解更多）
```

### 强制自检基线（V2.6 新增）

- 每次代码修改后，**必须**执行 self-check（28条规则 + 步骤5.5阻断点 + 步骤6实测）
- 步骤6（测试）**不可跳过**——纸面检查通过不算自检完成
- 首次加载 skill / 首次初始化项目后，**必须**执行一次完整的 baseline 自检，建立项目质量基线

### 项目状态检测触发逻辑（V2.6 新增）

每次 skill 激活时，先检测项目状态：

1. 检查项目根目录是否有代码文件（.js/.html/.py/.ts 等）→ 无 → 全新项目（路径A）
2. 检查是否有 `开发文档/` 目录 → 无 → 存量无文档（路径C）
3. 检查是否有 `开发文档/00-必读/开发文档管理.md` → 有 → 存量有文档（路径B）
4. 都不满足 → 存量无文档（路径C）

### 项目专属 Skill 指向规则

- 初始化后，项目流水线文件位于 `.ai-pipeline/`
- 项目专属的自检规则在 `.ai-pipeline/self-check/references/`
- 全局 Skill 模板在 `~/.codebuddy/skills/`（只读，不要修改）
- 开发时使用全局 Skill，但自检规则读取项目专属副本
