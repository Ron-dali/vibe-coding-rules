## 阶段 0：资产整理

> **目标**：把 Skill 源码打包成可直接上传到 GitHub 的干净版本。
> **耗时**：30 分钟

### 步骤 0.1：确认命名

所有对外展示内容统一使用 **Vibe Coding Rules**。
源码中 Skill 文件夹名保持不变（`pipeline-init` / `karpathy-principles` 等）——改文件名会破坏安装脚本。

### 步骤 0.2：创建开源发布目录

```powershell
# 在桌面创建开源发布目录
mkdir C:\Users\Administrator\Desktop\vibe-coding-rules -Force
cd C:\Users\Administrator\Desktop\vibe-coding-rules

# 创建子目录
mkdir pipeline-init, karpathy-principles, safe-terminal-executor, self-check, web-testing, changelog, .ai-pipeline -Force
```

### 步骤 0.3：复制 Skill 文件

```powershell
$src = "C:\Users\Administrator\Desktop\AI\Workbuddy\tbp\开发文档\开发者自创工具\AI编程流水线SKILL通用版"

# 复制 6 个 Skill
robocopy "$src\pipeline-init" "pipeline-init" /E /NFL /NDL /NJH /NJS
robocopy "$src\karpathy-principles" "karpathy-principles" /E /NFL /NDL /NJH /NJS
robocopy "$src\safe-terminal-executor" "safe-terminal-executor" /E /NFL /NDL /NJH /NJS
robocopy "$src\self-check" "self-check" /E /NFL /NDL /NJH /NJS
robocopy "$src\web-testing" "web-testing" /E /NFL /NDL /NJH /NJS
robocopy "$src\changelog" "changelog" /E /NFL /NDL /NJH /NJS

# 复制配套文件
copy "$src\pipeline.json" .
copy "$src\install.bat" .
copy "$src\install.ps1" .
copy "$src\install.sh" .
copy "$src\MEMORY_BOOTSTRAP.md" .
robocopy "$src\.ai-pipeline" ".ai-pipeline" /E /NFL /NDL /NJH /NJS
```

## 阶段 1：GitHub 上线

> **目标**：仓库公开可访问，有 README + LICENSE + 6 个 Skill。
> **耗时**：约 1 小时

### 步骤 1.1：注册 GitHub（如果还没有）

1. https://github.com/signup
2. 用户名建议：`laosi-ai` 或你的品牌名
3. 验证邮箱

### 步骤 1.2：确认 Git 已安装

```powershell
git --version
# 应返回 git version 2.xx.x
# 如果没有 → https://git-scm.com/download/win
```

### 步骤 1.3：创建 GitHub 仓库

1. https://github.com/new
2. 填写：

| 字段 | 填什么 |
|:--|:--|
| Repository name | `vibe-coding-rules` |
| Description | Vibe Coding needs Rules. Self-growing code quality pipeline for AI agents. |
| Public | ✅ 选 Public |
| Add README | ❌ 不勾（你有自己的） |
| .gitignore | 选 `Node` |
| License | ❌ 先不选（后面手工放） |

3. 点 **Create repository**

### 步骤 1.4：上传代码

```powershell
cd C:\Users\Administrator\Desktop\vibe-coding-rules

git init
git config user.name "[你的GitHub用户名]"
git config user.email "[你的GitHub邮箱]"

git add .
git commit -m "🎉 Initial release: Vibe Coding Rules V2.2"

git remote add origin https://github.com/[你的用户名]/vibe-coding-rules.git
git branch -M main
git push -u origin main
```

> 如果 git push 弹登录窗口：用 GitHub 账号登录。
> 如果弹出浏览器授权页：点 Authorize。

### 步骤 1.5：放 README 和 LICENSE

```powershell
# 用开源手册里的模板（先替换占位符）
copy "..\AI编程流水线SKILL通用版\开源手册\07-GitHub-README模板.md" "README.md" -Force

# 复制 Apache 2.0 LICENSE
copy "..\AI编程流水线SKILL通用版\开源手册\02-LICENSE模板-Apache2.0.txt" "LICENSE" -Force

git add README.md LICENSE
git commit -m "📄 Add README and Apache 2.0 LICENSE"
git push
```

### 步骤 1.6：仓库设置

1. 打开 `https://github.com/[你的用户名]/vibe-coding-rules/settings`
2. **Topics**：添加 `vibe-coding` `rules` `agent-skills` `ai-coding` `code-quality` `self-growing` `codebuddy` `cursor`
3. **Social preview**：上传一张封面图（可用 Canva 做一个 Logo）

### 步骤 2.1：确认 LICENSE

打开 `https://github.com/[你的用户名]/vibe-coding-rules/blob/main/LICENSE`，确认 Apache 2.0 全文可见。

> **Apache 2.0 保护什么**：别人用你的代码必须保留版权声明；不能拿你的方法申请专利告你；修改后可不开源。

### 步骤 2.2：确认 README 署名

README 底部应有：`© 2026 [你的名字]. Licensed under Apache 2.0`

## 附录 B：常见问题

**Q：GitHub 用户名能改吗？**
A：能，Settings → Account → Change username。但旧链接会失效，所以第一次起好。

**Q：公开仓库别人能偷我的代码吗？**
A：Apache 2.0 允许使用，但必须保留你的署名和 LICENSE。不署名=侵权。

**Q：如果一直没人 Star 怎么办？**
A：正常。开源前 3 个月关注度≈你自己的推广力度。不是你发了就有人看——阶段 3 就是在做主动推广。

**Q：Vibe Coding 是 Karpathy 的词，我能用吗？**
A：能。你这是对 Vibe Coding 的补充和改进，不是冒用。Apache 2.0 保护你的代码，不限制你对文化现象的引用和评论。

**Q：SKILL 文件的 TBP 项目引用要不要清理？**
A：不用。TBP 是你的"故事来源"——"来自真实项目实践"本身就是卖点。开源用户看到的是通用版，V2.0 已经做过去耦重构。
