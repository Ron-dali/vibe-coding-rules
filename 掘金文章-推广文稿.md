# AI 写代码最大的 Bug，是它没有记忆——我给 AI 装了个"外置大脑"

> 代码小白，用 AI 写了 14 万行代码。被同一个 Bug 坑了 10 次之后，我做了一件事：教 AI 自己记住坑在哪。

---

## AI 不记打，才是最致命的 Bug

用 AI 写代码的人都有过这种体验——

你发现了一个 Bug，花半小时修好。过两天，换个文件，AI **又写出来了，一模一样的。**

```
// 这个 Bug 我明明上个月就修过了！
if (count || 0) { ... }   // 0 被当 false 吞了，排查半小时
```

不是 AI 笨。是**每次对话都是独立的**——它不知道昨天的自己犯过什么错。

改了聊天窗口，工作流编辑器炸了——因为底层共享了同一个 DOM 容器。AI 只管当前文件，不管耦合关系。

传统 lint 工具能解决这个问题吗？不能。它们只能检查固定的语法规则，不会"记住"你踩过的坑。

**这不是代码质量问题，是"记忆"问题。**

---

## 我做了件很笨的事

我，编程小白。不会写代码。全靠 AI 写了 14 万行，做了个三端平台。

被 Bug 反复坑了无数次之后，我做了一个决定：

**我不学编程，我教 AI 守规矩。**

我把每次踩坑的经验写成"编程纪律"，变成一个 Skill 包。AI 每次改代码前自动自查，改完自动扫描，同一个坑绝不犯第三次。

两个月后，这套纪律自己"活"了。

---

## 三个别人没有的东西

### ① 🍞 面包屑系统——AI 终于有"记忆"了

文件改完那一刻，自动在文件头部种下一颗"面包屑"：

```
// 🍞 COUPLED: history.js (shared DB table), task-chat.js (截断点)
// 🍞 GOTCHA: 容器重建时必须重置 _rendered 标记，否则 DOM 假活
```

下次 AI 动 `history.js`，这颗面包屑会跳出来：**"注意！你跟 chat.js 共用了一张表，改字段时两边要对齐。"**

这才是真正的外置记忆。不是写在 prompt 里的静态提示——是**长在代码里、跟文件绑定、AI 自动读取的**。

### ② 信任分级规则自生长——Bug 变铁律

```
Tier 0 观察池：新发现的坑，先记下来观察
Tier 1 软规则：第 2 次出现，AI 改代码时会提醒
Tier 2 硬规则：第 3 次出现，AI 绝对不能犯，强制拦截
```

这不是硬编码的规则表。它会**自己长**。

今天 `|| 0` 吞值被坑了，记进观察池。下次再犯，升软规则。第三次——对不起，AI 在动手之前就会被拦住。

**你不需要永远记住每个坑。你的 AI 会帮你记住。**

### ③ 完整闭环——改代码像走流水线

不是"给几个提示就完了"。是一条真正的流水线：

```
改前：五原则自查 + 陷阱拦截
  ↓
安全执行：命令封装 + 超时保护  
  ↓
改后：28 条规则扫描 + 面包屑播种
  ↓
自动测试：浏览器模拟 + OCR 截图验证
  ↓
变更日志 + 生长检测
  ↓
✅ 闭环交付
```

装完之后，AI 的"犯错空间"被一层层收紧。不是靠人盯着，是靠自动化的防火墙。

## 为什么跟别的 lint 工具不一样？

| | ESLint / Prettier | vibe-coding-rules |
|:--|:--|:--|
| 规则来源 | 人工预设，写死的 | 从你的真实 Bug 中长出来的 |
| 耦合感知 | 无 | 🍞 面包屑自动标记文件关系 |
| 进化能力 | 需要人手动加规则 | 同一个 Bug 犯 3 次自动升硬规则 |
| 覆盖范围 | 仅代码风格 | 改前→改中→改后→测试→日志，全流程 |
| 适用对象 | 程序员自查 | AI 自动执行，不需要你懂代码 |

**传统工具是编程规范，这个是 AI 编程纪律。**
