# 历史事件模式速查表

> 以下为通用 bug 模式，摘自真实项目数据。新项目可在 pipeline.json 中追加自己的事件。

## 模式1：工具注册遗漏
- **教训**：自检必须做全集差集比对，不凭记忆
- **模式**：新增工具后只关注新工具，不做全量 diff → 旧工具遗漏

## 模式2：数据传输截断
- **教训**：自检必须覆盖数据传输链路的截断点
- **模式**：用小数据测试→正常假象→大数据被截断→静默丢失

## 模式3：自动保存覆盖真实数据
- **教训**：beforeunload 不可发送完整 payload；后端 upsert 不可硬编码字段
- **模式**：前端硬编码 + 后端硬编码叠加 → 真实值被覆盖

## 模式4：`||` 吞 0 导致数据不一致
- **教训**：数值字段必须用 `!= null`，不能用 `||`；select 必须有占位 option
- **模式**：`value || default` 吞掉 0/''/false

## 模式5：浮层/弹窗缺少关闭机制
- **教训**：浮层必须穷举关闭路径（X/ESC/遮罩/导航离开）
- **模式**：只考虑"怎么打开"，没考虑"怎么关闭"

## 模式6：路由注册顺序劫持
- **教训**：固定路径必须注册在参数化路径之前
- **模式**：`/api/items/:id` 劫持 `/api/items/search`

## 模式7：前端显示名映射遗漏
- **教训**：新增工具 = 注册层 + 前端 UI 显示名
- **模式**：只检查注册层，遗漏前端显示名 map

## 模式8：布局迁移遮罩残留
- **教训**：布局模式迁移时旧工件必须清零
- **模式**：遮罩层/position/z-index 未清理

## 模式9：同类组件功能不对等
- **教训**：同类组件必须功能对等
- **模式**：升级一处，遗漏同类组件

## 模式10：容器重建子组件假活
- **教训**：innerHTML 重建容器前必须重置所有子组件标记
- **模式**：DOM 已销毁→标记未清零→代码以为还活着

## 模式11：快照恢复数据污染
- **教训**：快照恢复必须分离"无关历史"与"相关上下文"
- **模式**：旧上下文数据从快照恢复到新场景→污染

## 模式12：多列表选中状态互斥
- **教训**：共享"选中项"语义的列表必须统一状态管理
- **模式**：各列表独立维护状态→互斥逻辑缺失

## 模式13：双入口不同步
- **教训**：双入口不止同步 script，还要同步 DOM
- **模式**：一处新增 DOM → 另一处未同步
