# 🏷️ 命名策略与品牌指南 — Vibe Coding Rules

## 为什么叫 Vibe Coding Rules

### 背景

2025 年，Andrej Karpathy（前 Tesla AI 总监、OpenAI 联合创始人）创造了一个词：**"Vibe Coding"**

> "There's a new kind of coding I call 'vibe coding'... I fully give in to the vibes, embrace exponentials, and forget that the code even exists."
> — Andrej Karpathy, Feb 2025

"Vibe Coding"描述的是 AI 编程的新范式：跟着感觉走，让 AI 写代码，你几乎忘了代码本身的存在。

### 我们的定位

**Vibe Coding Rules** 是 Vibe Coding 的对立面——不是反对，而是**互补**。

> Vibe Coding 给你速度，Rules 给你安全感。

Karpathy 说的"embrace exponentials"是对的——但 exponentials 需要方向盘。

### Slogan 体系

```
主 Slogan：  Vibe Coding needs Rules.
中文 Slogan：让 AI 守纪律，不是靠祈祷，是靠工程手段。
英文 Slogan：Discipline for AI, not by prayer, but by engineering.
```

### 命名金字塔

| 层级 | 用名 | 场景 |
|:--|:--|:--|
| **品牌名** | Vibe Coding Rules | GitHub、agentskills.io、社交媒体 |
| **中文名** | AI编程纪律 | 知乎、微信群、中文社区 |
| **简称** | VibeRules / VCR | 日常对话、标签 |
| **技术名** | AI Programming Pipeline SKILL | 内部文档、pipeline.json |

### 为什么这个名字能赢

1. **自带流量** — "Vibe Coding"在 Twitter/X、Hacker News、知乎上已积累海量讨论。你的名字天然命中这个搜索词
2. **反差感** — "Vibe"（随心所欲）vs "Rules"（规矩），一个名字就讲完了你的产品故事
3. **双关** — "Rules" 既是"规则"也是"很棒"（slang: "this rules!"），英文用户会心一笑
4. **中英通吃** — "Vibe Coding Rules" 中文用户也能读，不需要翻译就能理解是"给感觉写码定规矩"
