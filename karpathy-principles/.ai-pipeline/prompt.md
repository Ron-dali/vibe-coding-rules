# 强制激活：AI编程四大原则

> **每次编程指令必须执行原则自检。不依赖任何特定 IDE 或 Agent 平台。**

## 原则检查清单（输出代码前逐条过）

□ 原则1 — 先思考：真的理解问题了？确定改对文件了？先读文件确认现状。
□ 原则2 — 保持简单：能不能用最少改动解决？不引入不必要的抽象。
□ 原则3 — 精准修改：只改必须改的代码。不动无关逻辑。
□ 原则4 — 目标驱动：改完能跑吗？JS+CSS+模板同步了吗？
□ 陷阱A — 数值不用 || 判空（会吞 0/''/false），用 !=null 或 ??
□ 陷阱B — select 必须加占位 option，防止浏览器假选中
□ 陷阱C — POST upsert 更新分支从 req.body 取值，不硬编码字段

## 三大陷阱（通用编程模式）

| 陷阱 | 错误模式 | 正确写法 |
|------|---------|---------|
| JS 数值判空吞 0 | `value \|\| 1000` | `value != null ? value : 1000` |
| select 假选中 | `<select>` 无默认选项 | `<option value="" disabled selected>请选择</option>` |
| UPSERT 硬编码 | UPDATE 分支写死字段值 | 从请求体取值，fallback 到已有值 |
